# 用户档案更新

## 触发条件

counseling 对话结束后，由 counseling skill 或 Agent 判断是否调用本 skill。触发条件：
- 对话产生了新的诊断信息（维度变化、综合体变化、关键事件）
- 日常闲聊不触发

## 工作目录

用户档案位于 workspace 根目录下：`user_profile/`

## 流程

### 1. 判断是否值得更新

检查本次对话是否产生了以下任一：
- 某个维度/综合体的状态有实质性变化
- 发生了关键事件（转折点、突破、崩溃后反弹等）
- 形成了新规则或新认知框架
- 力比多方向或性质有变化

不满足则不更新，直接结束。

### 2. 检查周期更新

读 `user_profile/last-update.md`，获取上次更新的日期字符串（格式：YYYY-MM-DD）。

计算距今天数：
- ≥7 天：触发**周更新**——每个维度（dim1-dim3）和 comprehensive 写入 `{folder}/weekly/YYYY-WXX.md`（WXX 为 ISO 周数）
- ≥30 天：触发**月更新**——每个维度（dim1-dim3）和 comprehensive 写入 `{folder}/monthly/YYYY-MM.md`
- ≥90 天：触发**季更新**——每个维度（dim1-dim3）和 comprehensive 写入 `{folder}/quarterly/YYYY-QX.md`

周期更新文件内容：本周期内的关键变化总结 + 当前状态快照。

周期检查完成后，更新 `last-update.md` 为当天日期。

### 3. 更新 overview 文件

#### 3.1 综合档案：`user_profile/comprehensive/overview.md`

多时间尺度分层结构，详见文档末尾《各 Overview 文件编写规范》。

#### 3.2 根路由文件：`user_profile/overview.md`

薄层路由文件，仅含入口链接。无需更新。

#### 3.3 维度文件：`user_profile/four-dimensions/dim1-elements/overview.md`、`dim2-links/overview.md`、`dim3-karma/overview.md`

在本次对话中有显著变化时更新对应文件：
- 更新时间标记
- **当前快照**：修改变化的部分
- **关键变化**：追加新条目

### 4. 更新原则（适用于所有文件）

- **不覆盖历史**：overview 覆写是快照更新，周期文件是追加式记录，两者不冲突
- **引用原话**：关键变化描述尽量引用用户原话作为证据
- **简洁**：每条快照 1-2 句，不写论文
- **定性不定量**：不评分数，不打数字分。用"强/中/弱""通畅/阻滞""直接性/对抗性"等定性描述
- **不猜测**：只写对话中明确呈现的，不从"无记录"的日子强行推断
- **与用户自述一致**：用户说"心力不错"就不能写"心力低迷"，可在自述基础上补充分析视角但不能推翻

## 综合文件更新规范

综合文件横向识别维度间交互，与维度文件的纵向提取互补。

### 更新顺序

1. 更新 `four-dimensions/dim1-elements/` 三个周期文件
2. 更新 `four-dimensions/dim2-links/` 三个周期文件
3. 更新 `four-dimensions/dim3-karma/` 三个周期文件
4. 更新 `comprehensive/` 三个周期文件（最后——需参考维度文件结果写交互因果链）
5. 更新各个 overview 文件
6. 更新 `user_profile/last-update.md`

### 综合 weekly 文件的每日条目编写规则

**a. 用户自评（五维打分，百分制）**
- 行首固定格式：`用户自评: 身体XX / 体感XX / 心情XX / 心力XX / 觉察XX`
- 用户没打某项写 `—`，绝对不替用户补数字。同一天多次打分取最后一次

**b. 叙事**
- 只能从对话中提取，不能编造。一句话概括当日事件/状态/变化
- 当天无记录 → 写"（无记录）"；连续多天无记录 → 合并写"（无详细日记录）"

**c. 五综合体动态**
- 格式：`五综合体动态: 能量-xxx；力比多-xxx；涵容-xxx；感受-xxx；认知-xxx`
- 只写当日实际发生了显著变化的综合体，没变化不写。Agent 写定性文字评估，不打分
- 显著变化判定：对话中出现明确状态变化描述，或有可观察的行为改变

**d. 心灵维动态**
- 格式：`心灵维动态: xxx`，有显著变化才写
- 判定标准：出现存在感、自性光明、生死观、境界转换的明确表达

### 交互因果链提取规则

**规则1——跨维度门槛**：至少涉及五综合体、维度之间的交互因果。仅在单一综合体或维度内的变化不算交互链。

**规则2——时间先后+概念因果**：每个→须同时满足时间先后和概念因果，不能凭空构造箭头。每条链附"证据来源"（如"5.11晕倒记录+5.20金矿文章+5.29晨三个洞察"）。

**规则3——跨时间分析**：更新本周期内容时，加载上个周期（上周）及上个维度周期（上月）综合分析。

**规则4——影响力排序**：超过3条时优先保留涉及重大课题的、产生具体行为改变的、涉及综合体多的。跨月延续的链每月更新到最新节点。

**链的格式**：
```
### 核心链X：[方向概括]
[事件链，用→连接，括号标注涉及的综合体、维度]
方向：[因果方向，一句话]
为什么是核心链：[理由]
证据来源：[简注]
```

## 各 Overview 文件编写规范

具体模板详见同目录下的 `_overview-templates.md`。

### 更新规则

- **周更新**：追加周度变化条目，更新最近关键变化列表
- **月更新**：追加月度变化条目，下层变化中有结构性意义的向上传导
- **季更新**：追加季度变化条目
- **年度更新后**：更新"近三年状态"——重新评估三年窗口内的主要矛盾和变化
- 上层周期（年/季）只在有结构性变化时修订，不机械每周重写
- 下层周期的变化若构成上层周期的结构性变化，向上传导
