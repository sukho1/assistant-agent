# 用户档案更新

## 触发条件

counseling 对话结束后，由 counseling skill 或 Agent 判断是否调用本 skill。触发条件：
- 对话产生了新的诊断信息（维度变化、综合体变化、关键事件）
- 日常闲聊不触发

## 工作目录

用户档案位于 workspace 根目录下：`user_profile/`

## 流程

### 1. 判断是否值得更新

检查本次对话是否产生了以下任一：
- 某个维度/综合体的状态有实质性变化
- 发生了关键事件（转折点、突破、崩溃后反弹等）
- 形成了新规则或新认知框架
- 力比多方向或性质有变化

不满足则不更新，直接结束。

### 2. 检查周期更新

读 `user_profile/last-update.md`，获取上次更新的日期字符串（格式：YYYY-MM-DD）。

计算距今天数：
- ≥7 天：触发**周更新**——每个维度（dim1-dim3）和 comprehensive 写入 `{folder}/weekly/YYYY-WXX.md`（WXX 为 ISO 周数）
- ≥30 天：触发**月更新**——每个维度（dim1-dim3）和 comprehensive 写入 `{folder}/monthly/YYYY-MM.md`
- ≥90 天：触发**季更新**——每个维度（dim1-dim3）和 comprehensive 写入 `{folder}/quarterly/YYYY-QX.md`

周期更新文件内容：本周期内的关键变化总结 + 当前状态快照。

周期检查完成后，更新 `last-update.md` 为当天日期。

### 3. 更新 overview 文件

#### 3.1 综合档案：`user_profile/comprehensive/overview.md`

多时间尺度分层结构，详见文档末尾《各 Overview 文件编写规范》。

#### 3.2 根路由文件：`user_profile/overview.md`

根目录 overview.md 为薄层路由文件，仅含入口链接。无需更新。

#### 3.3 维度文件：`user_profile/four-dimensions/dim1-elements/overview.md`、`dim2-links/overview.md`、`dim3-karma/overview.md`

保留前三维。在本次对话中有显著变化时更新对应文件：
- 更新时间标记
- **当前快照**：修改变化的部分
- **关键变化**：追加新条目

### 4. 更新原则

- **不覆盖历史**：overview 覆写是"快照更新"，周期文件是追加式记录。两者不冲突。
- **引用原话**：关键变化描述尽量引用用户原话作为证据。
- **简洁**：每条快照 1-2 句，不写论文。
- **定性不定量**：不评分数。用"强/中/弱""通畅/阻滞""直接性/对抗性"等定性描述。
- **不猜测**：只写对话中明确呈现的，不推断"可能""也许"。

### 5. 分布式加载

counseling 启动时只读 `user_profile/comprehensive/overview.md`。分析到具体维度时，按需读 `four-dimensions/dim1-elements/overview.md`、`dim2-links/overview.md`、`dim3-karma/overview.md`。周期文件只在需要历史轨迹时才加载。

## 综合文件更新规范

综合文件的更新逻辑与维度文件有本质区别：维度文件是按领域纵向提取，综合文件是横向识别维度间交互。

### 更新顺序

update 时按以下顺序执行，不可打乱：

1. 更新 `four-dimensions/dim1-elements/` 三个周期文件
2. 更新 `four-dimensions/dim2-links/` 三个周期文件
3. 更新 `four-dimensions/dim3-karma/` 三个周期文件
4. 更新 `comprehensive/` 三个周期文件（最后）
5. 更新各个overview文件
6. 更新 `user_profile/last-update.md`

原因：综合文件需要参考维度文件的更新结果来写交互因果链，所以必须最后写。

### 综合 weekly 文件的每日条目编写规则

**a. 用户自评：五维打分（身体/体感/心情/心力/觉察，百分制）**
- 行首格式固定为 `用户自评: 身体XX / 体感XX / 心情XX / 心力XX / 觉察XX`
- 从用户对话中解析。解析模式：识别五个维度情况（以及可能的同义词如"状态"对应身体、"感受"对应心情等，但优先匹配原文用词）
- 用户没打某项就写 `—`。绝对不替用户补数字
- 多个同一天内多次打分，取最后一次

**b. 叙事**
- 只能从用户的对话消息中提取，不能编造
- 一句话概括当日发生的主要事件/状态/心态认知变化
- 用户当天无发言且无日记记录 → 写"（无记录）"
- 对无记录的连续多天（如 W20 的 5.13-5.17），合并为一个条目，写"（无详细日记录）"

**c. 五综合体动态**
- 格式：`五综合体动态: 能量-xxx；力比多-xxx；涵容-xxx；感受-xxx；认知-xxx`
- 用 `; ` 分隔不同综合体
- 只写当日**实际发生了显著变化**的综合体，没有变化的不写
- Agent 写的是**定性文字评估**，不是分数：
  - ✓ "力比多-投注错位被识别，对抗性被身体事实动摇"
  - ✗ "力比多7/10"（不要打分）
  - ✗ "力比多：正常"（没变化就不写）
- "显著变化"的判定标准：对话中出现了对该综合体状态变化的明确描述（如"不装了"、"对抗不再需要了"、"睡饱了脑袋像野驴"）；或者行为上出现了可观察的改变

**d. 心灵维动态**
- 格式：`心灵维动态: xxx`
- 同上，有显著变化才写
- 心灵维动态的判定标准：出现涉及存在感、自性光明、生死观、境界转换的明确表达（如"要我死就死"、"从书生到战士"、"看清不消沉"）

### 周度、月度、季度、年度文件中都需要有细分条目变化变化
五综合体、前三个维度的各自变化变化

### 周度、月度、季度、年度文件中都需要有交互因果链
提取规则如下：

**规则1——跨维度门槛**：必须至少涉及五综合体、四个维度之间的交互因果，才算"交互链"。仅在单一综合体或维度内的变化（如"本周社交方面逐渐建立渠道"）不算交互链，归到"五综合体月度变化"对应条目。

**规则2——时间先后+概念因果**：链中每个→必须同时满足：(a) A 发生时间在 B 之前，(b) 用户对话或日记中有证据表明 A 是 B 的原因。间接证据可接受，但不能凭空构造箭头。做证据笔记：每条链最后附一个"证据来源"行，简要标注从哪些对话中提取（如"5.11晕倒记录+5.20金矿文章+5.29晨三个洞察"）。

**规则3——跨时间分析**：注意捕捉连续的时间周期下，交互因果的链条，比如本周/月/季/年的变化和改变，跟本周/月/季/年的动态的关系，在更新本周期的内容时（比如更新本周内容时），要加载上个周期（对应上周），及上个维度的周期（对应上月）的内容综合分析。

**规则4——影响力排序**：超过3条时按以下标准取舍：
- 涉及重大课题的
- 产生了具体行为改变的优先于只有认知改变的
- 涉及综合体数量越多越优先
- 跨月延续的链每月更新到最新节点，不照抄上月

**链的格式（固定）**：
```
### 核心链X：[方向概括]
[事件链，用→连接，括号标注涉及的综合体、维度]
方向：[因果方向，一句话]
为什么是核心链：[理由]
证据来源：[简注]
```

### Agent 对综合状态的文本评估原则

- Agent 不打数字分。Agent 只做定性文本评估（"五综合体动态"和"心灵维动态"）
- Agent 不从"无记录"的日子强行推断。连续多天无记录的，只写"（无详细日记录）"
- Agent 的评估与用户的自我描述保持一致。用户说"心力不错"，Agent 就不能写"心力低迷"。可以在用户自述的基础上补充分析视角，但不能推翻

## 各 Overview 文件编写规范

每个 overview 文件按时间尺度分层，从上到下由粗到细。每层包含两个部分：**主要矛盾**（当前周期最核心的课题，1-2句）和**主要变化**（该周期发生的关键变化，1-2句）。

具体模板详见同目录下的 `_overview-templates.md`。

### 更新规则

- **周更新**：追加周度变化条目，更新最近关键变化列表
- **月更新**：追加月度变化条目，下层周度变化中有结构性意义的向上传导
- **季更新**：追加季度变化条目
- **年度更新后**：更新"近三年状态"——重新评估三年窗口内的主要矛盾和变化
- 上层周期（年/季）只在有结构性变化时修订，不机械每周重写
- 每条主要矛盾 1-2 句，主要变化 1-2 句
- 下层周期的变化如果构成了上层周期的结构性变化，向上传导
- 引用原话，不定量打分，不猜测
