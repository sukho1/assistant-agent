# 周期文件格式规范

> 事实与分析分离的核心规则见 SKILL.md。以下为周期文件特有格式细节。

## 事实编号规范

- 每条事实必须标注具体日期（日或时间点），格式：`[日期] [编号] [事实描述]`
- 周记中按日编号：`[日序].[事实序]`，如 `1.1`（周一第1条）、`3.2`（周三第2条）；子项 `1.1.a`、`1.1.b`
- 月记/季记摘录时保留日期和原编号：`5.25 1.1. [事实简述]`，跨文件引用仍可追溯

---

## 文件类型

三种周期文件，均为**追加式记录**（每次更新在末尾添加新条目，不覆写已有内容）：

- `weekly/YYYY-WXX.md` —— 周度记录，按日追加事实，并更新本周总结
- `monthly/YYYY-MM.md` —— 月度汇总，摘录重要事实
- `quarterly/YYYY-QX.md` —— 季度汇总

四个位置各有一份：`comprehensive/`、`four-dimensions/dim1-elements/`、`dim2-links/`、`dim3-karma/`。

## 一、综合周期文件格式

### 1.1 综合周记（`comprehensive/weekly/*.md`）

```
# 综合周记 WXX (日期范围)

## 用户事实

### YYYY-MM-DD (星期X)
1.1. 用户自评: 身体XX / 体感XX / 心情XX / 心力XX / 觉察XX
1.2. 身体: [状态描述]
1.3. 社交: [关键互动]
1.4. [其他当日关键事件]

### YYYY-MM-DD (星期X)
2.1. ...
...

---
## Agent 分析

### 单日分析

#### YYYY-MM-DD (星期X)
- 五综合体: xxx（当日有变化才写，可引用当日事实编号）
- 心灵维: xxx（当日有变化才写）

#### YYYY-MM-DD (星期X)
- 五综合体: xxx
- （当日无显著心理变化则不写分析）

---
### 本周综合分析

#### 单日分析

#### 五综合体动态
能量-xxx；力比多-xxx；涵容-xxx；感受-xxx；认知-xxx
（本周显著变化的综合体，定性不打分）

#### 心灵维动态
xxx（有显著变化才写）

#### 本周综合位移
[本周关键变化的综合分析，可引用事实编号]

#### 交互因果链
[如有跨维度因果链]

### 核心链X：[方向概括]
[事件链，用→连接]
方向: [因果方向，一句话]
为什么是核心链: [理由]
证据来源: [简注]
```

**事实区规则**：
- 未打分维度写 `—`，绝不替用户补数字。同天多次打分取最后一次
- 当天无记录 → 写"（无记录）"；连续多天无记录 → 合并"（无详细日记录）"
- 新事实追加在当天事实区末尾，不修改已有条目

**分析区规则**：
- 分析也分两层：单日分析（与事实区日期对齐）+ 本周综合分析
- 单日分析只写当日有显著变化的维度，无变化则留空或省略
- 引用事实时用编号简述：`引用事实 1.2 用户与父亲通话后情绪波动`
- 避免长篇复述事实——引用编号即可，分析是增量价值

### 1.2 综合月记（`comprehensive/monthly/*.md`）

```
# 综合月记 YYYY-MM

## 用户事实
（从该月周记中摘录最重要事实，每条标注具体日期，保留原始编号）
5.25 1.1. [关键事件简述]
5.27 3.2. [重要状态变化]
...

---
## Agent 分析

### 月度交互因果链
### 核心链一：[方向概括]
...

### 五综合体月度位移
感受-xxx
认知-xxx
涵容-xxx
能量-xxx
力比多-xxx

### 心灵维月度位移
xxx
```

### 1.3 综合季记（`comprehensive/quarterly/*.md`）

格式同月记，时间尺度为季度。摘录该季最重要的事实，保留原始编号。

## 二、维度周期文件格式

三个维度的周期文件共用统一骨架，仅事实区字段不同。

### 周记模板

```
## 用户事实

### YYYY-MM-DD (星期X)
{维度事实字段}

---
## Agent 分析

### 单日分析

#### YYYY-MM-DD (星期X)
- [当日变化，引用事实编号]

---
### 本周综合分析

#### {维度名}周度位移
[本周变化总结]
```

### 各维度事实字段

| 维度 | 事实字段 | 周位移标题 |
|------|---------|-----------|
| dim1·五要素 | 1.1.学业 1.2.事业 1.3.社交 1.4.身体 1.5.心理心灵 | 五要素周度位移 |
| dim2·链接 | 1.1.自己 1.2.他人 1.3.社会 1.4.历史 1.5.自然宇宙 1.6.学业事业 | 链接周度位移 |
| dim3·业障 | 1.1.内在业障 1.2.外在业障 | 业障周度位移 |

月记/季记：只列该周期内发生显著变化的字段，保留原编号。

## 二b、历史材料的多时间层规则

历史材料（年末总结、回溯分析等）通常包含**两层时间**：
- **被描述层**：材料在讲哪个年份/月份的事（如"2023年"）
- **观察层**：用户是在什么时间点做的这些回看/反思（如"2025.7.10发现..."）

**两层都是事实，都必须写入对应周期文件。** 观察层的每个具体日期，是当日的重要心理事件，属于该日所在周的周记事实。

操作顺序：
1. 提取材料中**所有**显式日期（含年份的都要提取，写进待办清单）
2. 识别每个日期的"被描述对象"和"观察内容"（当天产生了什么洞察/情绪/命名）
3. **先处理观察层**：对每个观察日期，检查对应周记是否存在——存在则追加事实，不存在则新建
4. **再处理被描述层**：创建/更新被描述年份的周期文件，在事实区引用观察层日期

观察层默认只建 comprehensive 周记，dim 文件仅在该维度有具体内容时才建。

示例：用户提供"2023年末素描"，包含2025.7.10/11.28/12.13三个观察日期。错误做法是只创建2023年记；正确做法是先把这三个日期的内容写入对应周记（W28/W48/W50），再创建2023年记。

## 二c、入库筛选原则

profile 是心理动力学的骨架，不是日记摘要。并非所有内容都入库。

**入库标准**（满足任一）：
- 用户产生新的命名、洞察、认知框架（命名=结构性变化）
- 力比多方向或性质有变化
- 涵容发生了可验证的扩容或触底
- 业障模式首次被识别或明显消退
- 链接发生质变（新链接建立/旧纠缠切断/链接方式改变）
- 跨周以上后果的关键事件

**不入库**：
- 单日情绪起伏、常规社交、重复性反思
- 没有产生新认知的日常流水
- 在周记层面就会被合并掉的波动

**自检**：这条在月记提取时还会不会出现？不会出现就不入。

## 三、交互因果链提取规则

仅适用于 `comprehensive/` 周期文件。

### 规则1：跨维度门槛
至少涉及五综合体、维度之间的交互因果。仅在单一综合体或维度内的变化不算交互链。

### 规则2：时间先后 + 概念因果
每个→须同时满足时间先后和概念因果，不能凭空构造箭头。每条链附"证据来源"（如"5.11晕倒记录+5.20金矿文章+5.29晨三个洞察"）。

### 规则3：跨时间分析（跨周期交互因果链）

写任一周期文件的综合分析区时，必须加载两个参考周期的综合分析区：上一个同级周期的综合分析 + 上一个上级周期的综合分析。

| 当前写 | 参考1: 上个同级 | 参考2: 上个上级 |
|--------|----------------|-----------------|
| 周记 | 上周周记综合分析 | 上月月记综合分析 |
| 月记 | 上月月记综合分析 | 上季季记综合分析 |
| 季记 | 上季季记综合分析 | 上年年记综合分析 |
| 年记 | 上年年记综合分析 | —（近三年 overview 已含） |

参考目的：
- 识别**跨周期延续的链**：从上个周期延续到本周期的因果链，标注延续节点的变化（加剧/衰减/转化）
- 识别**新涌现的链**：本周期新出现的交互因果链
- 识别**已衰减/解决的链**：上周期存在但本周期不再活跃，判断是已解决还是被新矛盾覆盖

跨周期链在分析区单独标注，如"（跨周期延续：上月在核心链二中首次出现，本月该链…）"。

**执行步骤**（强制）：
1. 打开上个同级周期的 Agent 分析区，列出其所有核心链
2. 逐条判断：延续到本期（标注节点变化）？新涌现？已衰减/解决？
3. 打开上个上级周期的 Agent 分析区，用同上步骤判断上级链的跨级延续
4. 将跨周期判断结果写入本期分析区的交互因果链末尾

### 规则4：影响力排序
超过3条时优先保留涉及重大课题的、产生具体行为改变的、涉及综合体多的。跨月延续的链每月更新到最新节点。

### 链的格式
```
### 核心链X：[方向概括]
[事件链，用→连接，括号标注涉及的综合体、维度]
方向：[因果方向，一句话]
为什么是核心链：[理由]
证据来源：[简注]
```
