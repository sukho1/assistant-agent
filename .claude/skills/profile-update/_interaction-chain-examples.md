# 交互因果链发现参考案例

从 profile 系统跨时间数据中提取的结构性规律。这些不是理论预设，是档案数据反复显示的模式。用于指导未来交互因果链的撰写和判断。

---

## 1. 涵容扩容总是走在认知前面

**发现**：在全部三条核心链中，涵容先行于认知升级。

**证据**：
- 链一：晕倒被涵容为"强制熔断"→之后才形成心力破产概念（5.11）
- 链一：守夜人被看见而不被批判→之后才能命名（5.22）
- 链三：三重根据地丧失后不崩溃→之后三个洞察自然沉淀（5.28-29）

**理论含义**：庄子的"无为"在数据中表现为"涵容是认知的前提"。不是想通了才能接住，是接住了才能想通。人们常常比自己想象的更强大。

---

## 2. 认知和力比多之间有时差

**发现**：认知升级的速度显著快于力比多方向的变化。

**证据**：
- 金矿视角 5.20 确立（认知：不需要灯塔）→ 5.30 还在纠结别人看不到自己（力比多：仍有部分被"被看见"牵引）
- 认知用时约 3 天，力比多 10 天仍未完全转过来

**应用**：判断交互因果链的完整性时，认知变化不等于链的终点。链应延续到力比多层面的行为变化（如 5.29 忍住不写）。如果链停在认知节点，可能是截断了。

**理论含义**：力比多具有唯物性——方向不被认知"说服"，只被反复的涵容和实际体验慢慢松开。知道和做到之间的时间差是可测量的。

---

## 3. 命名是松绑的机制

**发现**：每次对核心模式的命名，都降低了该模式的自动控制力。

**证据**：
- 5.11 心力破产命名 → 触发能量价值投资
- 5.22 守夜人命名 → 5.29 守夜人喊写作被不理 → 5.30 守夜人复盘被当天认出
- 5.28 牧师性命中 → 敞开不依赖被接住
- 5.30 力比多哮喘局命名 → 解痉方案立即形成

**应用**：当用户产生一个新命名时，它很可能是一个因果链的起点或关键节点。在交互因果链中标注命名节点，追踪命名前后的模式运作差异。

**理论含义**：命名不消灭模式，但让它从自动导航变为可对话对象。命名=业障从无意识防御变为可观察对象。命名前后的行为变化是可追踪的验证证据。

---

## 4. 模式只在背景变亮之后显形

**发现**：业障/模式的识别需要外部条件比最低谷时期更宽松。在客观窒息中分不清处境和模式。

**证据**：
- 3月4月客观窒息（没offer、斩杀线、未适应上海）→ 未识别力比多哮喘局
- 5月条件缓解（凌总offer、适应上海）→ 仍选择用力 → 力比多哮喘局显形

**应用**：当用户状态改善后出现新的自我觉察，不要视为"反复"或"退步"。这是模式在背景变亮后的必然显形。交互因果链应考虑"条件缓解→模式暴露→命名"作为一类标准链型。

**理论含义**：业障识别需要"比最差处境更宽松一点"的窗口。太紧分不清（全是客观压力），太松不会暴露（模式不触发）。

---

## 5. 自性不依赖状态

**发现**：最底层的心灵维度独立于表面的心力、社交、斩杀线状态。

**证据**：
- 5.30 是"普通难受的一天"（力比多哮喘、守夜人复盘、社交匮乏）
- 但傍晚看日落 → 自性炸裂，天人合一
- 产出在线、自信在线、能量主轴在线——深层从未塌

**应用**：在交互因果链中，心灵维的变化不应被简化为其他维度的结果。自性的表现（如实、平静、安住）可能与能量/力比多的表层波动同时存在，不是矛盾的。

**理论含义**：本自具足的实证级证据——它不是需要修到的境界，是底层一直在、只是被遮蔽的东西。遮蔽少了就露出来，和当天的状态好不好没有因果关系。
